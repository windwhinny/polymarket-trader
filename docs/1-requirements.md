## 设计决策

### ADR-1: Agent 模式 vs 管道模式

**决策**: Agent 自主调用工具，而非预先收集数据喂给模型。

**理由**: 模型能自主决定搜什么、搜几次、何时停。这更接近真实交易——你不会等着别人把所有信息整理好，而是自己主动调研。

**后果**: Agent 可能做无效搜索（浪费 API 调用），但灵活性远高于管道模式。

### ADR-2: 市场日期过滤策略

**决策**: 使用 `endDate > month_end` 过滤，配合 `order=volume` + `closed=true` 获取市场。

**理由**: 避免在已结算市场上下注。每月自动减少可交易市场（结算后的市场消失）。

**问题**: 2025 年可用市场太少（Top 200 高成交量市场全是 2026 年的）。切换到 2026 Q1 后市场多样性明显改善。

### ADR-3: 搜索 API 选择

**决策**: 使用 SerpAPI（Google 搜索），替代 Tavily。

**理由**: SerpAPI 的 `tbs` 参数原生支持日期范围过滤，Tavily 的日期过滤时效性差。

### ADR-4: 手续费模型

**决策**: 模拟 3 层手续费：
- Taker fee: 0.01% (Polymarket 2024-2025 实际费率)
- Spread cost: 1% (bid-ask 差价的一半)
- Gas: $0.005 (Polygon, 2025 Q1 后可忽略)

### ADR-5: 单笔下注上限

**决策**: 单笔 ≤ 总资金 15%（代码限制），Agent prompt 建议 5-10%。

**理由**: 避免过度集中风险。Agent 偶尔会无视 prompt 建议，代码层面的硬限制更可靠。

### ADR-6: 月份独立性 vs 资金复利

**决策**: 按顺序跑，资金逐月复利。但第一阶段（数据拉取）4 线程并行。

**理由**: 真实交易中资金是连续的、复利的。并行数据拉取可以提速但 Agent 执行仍需串行（因为资金依赖上月结果）。
